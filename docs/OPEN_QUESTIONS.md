# Open Questions

Вопросы, обнаруженные при анализе PRD и требующие уточнения.

---

## Нерешённые (требуют ответа)

### Q1: Формат и спецификации файлов разных AI-агентов (Шаг 7)

PRD перечисляет агентов: Claude Code, ClaudeCLI, Gemini, GeminiCLI, Codex, Continue и др. Для каждого нужно создать «главный файл с правилами», отвечающий «спецификациям конкретного агента». Спецификации этих форматов не описаны.

**Решение отложено до Этапа 7 (Agent File Generators).** На тот момент будут проанализированы требования каждого агента.

---

### Q2: Именование выходного файла фреймворка

В `.agents/rules/` и CLAUDE.md выходной файл назван `frameworkname.md` буквально. Но логично называть его по имени выбранного фреймворка (например, `angular-guidelines.md`).

**Решение отложено до Этапа 6 (Compiler).** Предлагаемый вариант: выходной файл называется так же, как исходный файл фреймворка (без изменений). CLAUDE.md и другие агентские файлы ссылаются на актуальное имя.

---

## Решённые в ходе анализа

| Вопрос | Решение |
|---|---|
| Q3: `packages` vs `package` | Использовать `packages`. Переименовать существующие директории. |
| Q4: Формат конфига | `ai-rules-config.json` в корне проекта. Плоский JSON: `version`, `projectName`, `architecture`, `framework`, `packages[]`, `agents[]`, `lastSync`. |
| Q5: Удалённый режим | Два режима: локальный (`node path/to/index.js`) и удалённый (`npx github:user/repo`). Шаблоны включаются в npm-пакет через `"files": ["dist/", "context/"]`. Скрипт читает их относительно `import.meta.url` в обоих режимах. Для приватных репозиториев пользователь настраивает SSH-ключи. Никакого git clone в проекте пользователя. |
| Q6: Компиляция package-rules.md | Простая конкатенация содержимого выбранных файлов с заголовком `# Code Style & Tools`. Если ничего не выбрано — файл не создаётся, и в агентских файлах ссылка на него отсутствует. |
| Q7: Модификация `.gitignore` | Только вывести рекомендацию пользователю. Скрипт не модифицирует `.gitignore` самостоятельно (безопаснее). |
| Q8: Язык CLI | Все сообщения на английском. |
| Q9: Отсутствие `context/projects/<name>/rules/` | Если папка не найдена — просто продолжаем без проектных переопределений (не создаём автоматически). |
| Q10: «Пустой» файл | Файл считается пустым, если не существует ИЛИ его содержимое после `.trim()` имеет длину 0 байт. |
| Q11: Тип выбора архитектуры | Radio (строго один вариант): Frontend / Backend / Fullstack. Добавлена поддержка fullstack через `context/rules/fullstack/`. |
| Q12: Атомарность операций | Запись напрямую в `.agents/rules/`. При ошибке — сообщение пользователю. Транзакционность не реализуется (избыточно для данной задачи). |
| Q13: Overwrite/Append/Skip | Единая логика для **всех** агентских файлов (CLAUDE.md, AGENTS.md, .cursorrules, и др.). |
| Q14: `rules/projects/<name>/spec.md` | Копируется как есть в `.agents/rules/spec.md`. Не влияет на другие файлы. |
